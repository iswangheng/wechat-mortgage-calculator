# 微信云开发接入方案 - 数据自动更新

## 目标
将房贷数据（LPR利率、城市政策）从硬编码改为云端动态获取，实现数据更新不用发版。

## 技术方案

**架构：小程序 → 云数据库 → 云函数定时更新**

```
┌─────────────────────────────────────────────┐
│  小程序启动                                   │
│  ├─ 1. wx.cloud.init()                       │
│  ├─ 2. 从云数据库拉取最新数据                    │
│  ├─ 3. 成功 → 覆盖本地配置，缓存到 Storage       │
│  └─ 4. 失败 → 降级使用本地 cities-2026.js       │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  云函数 (updateLprData)                       │
│  ├─ 定时触发器：每月21日 10:00 自动执行           │
│  ├─ 抓取央行官网 LPR 数据                       │
│  └─ 写入云数据库                               │
└─────────────────────────────────────────────┘
```

## 任务拆解

### Phase 1: 基础设施搭建（代码层面，不依赖云环境）

**1.1 创建数据服务层** `src/services/data-service.js`
- 封装数据获取逻辑：优先云端 → 降级本地 → 缓存策略
- 所有页面通过此服务获取数据，不直接引用 cities-2026.js
- 缓存有效期 24 小时，避免频繁请求

**1.2 改造 app.js**
- 添加 `wx.cloud.init()` 初始化（带环境判断，未开通云开发时跳过）
- 启动时预加载云端数据

**1.3 改造页面数据获取**
- index.js: `getCityConfig()` → `DataService.getCityConfig()`
- compare.js: 同上
- lpr.js: LPR 数据从云端获取
- 改动最小化：只替换数据获取入口，不改页面逻辑

### Phase 2: 云函数 & 数据库初始化脚本

**2.1 创建云函数 `cloud/functions/updateLprData/`**
- 抓取央行官网 LPR 公告页面
- 解析最新 1年期/5年期 LPR
- 更新云数据库中的 LPR 记录
- 配置定时触发器：每月 21 日 10:00

**2.2 创建云函数 `cloud/functions/getLatestData/`**
- 小程序端调用此云函数获取最新数据
- 返回：LPR利率 + 城市配置 + 元信息

**2.3 创建数据库初始化脚本 `cloud/database/init-data.js`**
- 将当前 cities-2026.js 的数据转为 JSON
- 导入到云数据库的 `mortgage_config` 集合
- 一次性执行，后续通过云函数更新

### Phase 3: 用户开通云开发后的配置（文档）

**3.1 编写部署指南 `docs/deployment.md`**
- 注册小程序账号步骤
- 开通云开发步骤
- 导入初始数据步骤
- 部署云函数步骤
- 配置定时触发器步骤

## 风险点

1. **云开发未开通时的兼容**
   - 缓解：`data-service.js` 判断 `wx.cloud` 是否可用，不可用时直接用本地数据
   - 现有功能零影响

2. **LPR 爬虫失败（央行网站改版）**
   - 缓解：爬虫失败时不更新数据库，保留旧数据；发告警通知开发者

3. **云数据库免费额度**
   - 免费额度：5万次/月读写，2GB存储
   - 预估用量：日活1000 × 每次2次查询 = 6万/月，接近上限
   - 缓解：用 Storage 缓存 24h，实际云端请求量降至 < 1万/月

## 验收标准
- [ ] 云开发未开通时，现有功能完全不受影响
- [ ] 云开发开通后，小程序启动时自动从云端拉取最新数据
- [ ] 云端数据变更后，用户下次打开小程序即可看到最新利率
- [ ] 云函数定时触发器正常执行，自动更新 LPR
- [ ] 部署文档清晰，按步骤操作即可完成上线
